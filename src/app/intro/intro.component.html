Angular Development

In April 2018, after launching my first commercial client website built on WordPress, I set aside C# training to begin taking Angular courses and building apps. This app is the result of much of the work I've been doing since then, as well as coding experience learned in courses on C# and JavaScript.

This Application

This application is a portfolio and personal development reference tool. It's built with some styling inspiration from angular.io. As a portfolio, this app is also it's own most interesting feature because it's the most complex application that I've built from scratch without following a course for guidance, though some features and techniques from course apps are used.

This app remains an important training tool, and is subject to further development. As a result the code remains loaded with commentary from the development process. You can see the code at GitHub.

Technology

This app is built in Angular 6, using Firebase Realtime Database and AngularFire2 to retrieve the data and pass it into RxJS observables. Other features of the app include Bootstrap classes, a FormBuilder reactive form with Validators in an ngx-bootstrap modal for adding apps, and an RxJS BehaviorSubject.

Philosophy

In the spirit of effenciency and progress I have often taken the general approach of primarily making the code work, and secondarily making it better. There are likely several features of this app that I will like to improve upon as time allows. Similarly, all apps in this portfolio should be considered works in progress.

App Showcase

The app showcase is made up of one or more sections, demarcated by horizontal rule, which may feature any number of apps. This arrangement is primarily for grouping together multiple versions of the same app.

Sections

Sections are a class in this app with two properties: a number and an array of apps.

Featured Apps

Apps are a class in this app with several properties valued by form entries in the modal which is opened by clicking the "Add App" button. These values then populate the section in the selected position.

The Position Property

A fairly complex feature of this app relates to the featured apps' "position" property and several services in this app are dedicated to managing it. The app is designed for flexibility in app positioning, such that an app can be added to any section, either with other apps already in it, or creating a new section and shifting other apps out of the way with their positions being updated in the database. Likewise, any app can be edited to move from one positon, with or without other apps in it, to another position, with or without other apps in it, with each different scenario adding a layer of complexity to the attendant position shifting that will occur to the apps' position properties.

The "Positions" Dropdown

The "Positions" available in the modal dropdown are read from the positions of apps retrieved from the database, with one extra added or removed depending on whether or not a new section is requested. When editing to move an app from a section with no other apps in it, an extra position is not added to the dropdown since the move will leave a hole causing all apps down the line to be shifted up one position to fill it.

App Description <br> Tag

Carriage returns entered into the "App Description" <textarea> are replaced with <br> tags. The data is then bound to the innerHTML property of the <p> tag.
